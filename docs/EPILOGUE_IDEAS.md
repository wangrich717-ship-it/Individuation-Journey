# 终章改进建议（基于前面所有章节）

当前流程：序章 → 第一章 意象之海 → 第二章 阴影的邂逅 → 第三章 内在声音 → 炼金术祭坛（选曼陀罗+涂色）→ **终章**。

---

## 一、与前面章节的呼应

| 章节 | 玩家留下什么 | 终章可怎么用 |
|------|--------------|--------------|
| **序章** | 面具/身份、选择、可能命名的阴影倾向 | 终章开头可提「你放下的面具」「你选的路」 |
| **第一章** | 阴影命名（shadowNames）、情绪命名（emotionName）、整合体验 | 已有：曼陀罗基于 shadowName；可加一句提及「你曾命名的情绪」或「整合的对立面」 |
| **第二章** | 回声诗、印记、知识「理性是工具，灵魂是向导」 | 已有：knowledge 列表展示；可加一句「灵魂之乐 / 纹章」与曼陀罗的意象衔接 |
| **第三章** | 意象之海、对立与超越、象征、shadowName 等 | 终章「自性的圆满」可与第三章的「整合」「象征」呼应 |
| **祭坛** | 五件道具放置、选的曼陀罗、**涂色结果**（regionColors） | 终章目前用程序生成曼陀罗；可改为/叠加「你在祭坛涂的那幅」 |

---

## 二、具体改动建议（按优先级）

### 1. 用「祭坛涂色曼陀罗」作为终章主图（高）

- 在 `gameState` 中存：`selectedMandalaIndex`、`mandalaRegionColors`（或序列化后的涂色结果）。
- 祭坛完成时把当前选中的曼陀罗 id 与涂色写入 state，终章读取后直接渲染同一张曼陀罗（用现有 `ColorableMandala` 的 path + 颜色，或导出为图）。
- 效果：玩家在终章看到的是「自己刚涂的那幅」，而不是另一张程序生成的图，仪式感更强。

### 2. 文案分阶段呼应旅程（中）

- **第一段**：保持「场景融为一体」。
- **第二段**：保留「完整而非完美」，可加半句与第一章整合、第二章灵魂、第三章意象挂钩（例如「你命名过的阴影、听过的回声、见过的意象，都在这幅图里」）。
- **第三段**：已改为「你已在祭坛为曼陀罗涂上颜色；此处再为你呈现……」；若采用建议 1，可改为「这便是你在祭坛上选定的曼陀罗，你为它涂上的颜色即是你此刻的内心图景」。

### 3. 「旅途中的领悟」与章节对应（中）

- `knowledge` 列表可标出来源（如「第二章」「第三章」），或按章节分组展示，让玩家看到哪些是祭坛前就得到的领悟。

### 4. 终章曼陀罗的生成逻辑（若保留程序生成）（低）

- 除 shadowName、inventory 外，可把 `shadowNames`、`emotionName`、部分 `knowledge` 的摘要参与种子或配色，使「心理图景」更贴近整段旅程。

### 5. 下载与再体验（低）

- 若终章展示的是祭坛涂色曼陀罗，下载的即这幅；若仍展示程序生成的，可考虑「下载当前这幅」与「下载祭坛涂色那幅」两个入口（实现成本较高，可作后续扩展）。

---

## 三、小结

- **必做（已做）**：画完曼陀罗后进入终章、按钮改为「完成，进入终章」、章节导航与完成逻辑改为祭坛→终章。
- **推荐**：终章主图改为玩家在祭坛涂色的那幅曼陀罗（需把选中图案 + 涂色存进 gameState 并在终章读取渲染）。
- **可选**：文案与「领悟」列表更明确地呼应各章，程序生成曼陀罗的种子/配色更多引用前面章节数据。
